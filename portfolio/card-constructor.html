Introduction

Card Constructor allows users to create decks of Magic: The Gathering cards and lets users look up card-specific information, such as prices of cards, all the different sets of each card, and so on.

The main functionality revolves around deck creation, where users can select the cards they want to create a Magic: The Gathering deck. Charts are created from the information gathered by the cards that are chosen. The charts are generated via the Angular Chart plugin for AngularJS. Pie charts, donut charts, bar charts, and line charts are used in this application.

Initial Ideas

I wanted to get some experience using AngularJS, so I thought that a deck creation website would give me enough flexibility to experiment with Angular and it would give me insight into how to tie Angular together with the backend, mainly MongoDB and Express.

The EchoMTG and MagicTheGathering.io APIs were used to gather pricing data and card-specific data. The User and Deck models were used so that users could save their decks to their profile as well.

Final Concept

There is still work to be done, mainly re-factoring code and developing more tests to ensure things are working properly. The website still needs to be better-suited for mobile devices, since it was largely designed with desktop-usage in mind at first. Still, users can register, log in, create decks or look up information about specific cards, and load their saved decks and edit them if they choose to do so. My favorite part was creating all the different charts!
